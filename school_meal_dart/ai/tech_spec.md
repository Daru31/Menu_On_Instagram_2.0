# 프로젝트 기술 명세서 (Technical Specification)

## 1. 프로젝트 개요

- **프로젝트명**: School Meal Auto-Uploader (Flutter & GitHub Actions 기반)
- **목적**: 매일 정해진 시간에 학교 급식 정보(중식/석식)를 수집하고, Flutter의 강력한 UI 렌더링 기능을 활용해 인스타그램 감성에 맞는 고품질 이미지/영상으로 변환한 뒤, 인스타그램 계정에 자동 업로드하는 시스템을 구축한다.
- **특징**: 로컬 PC나 별도의 서버 구동 없이, **GitHub Actions**의 스케줄러를 활용하여 100% 클라우드 환경에서 자동화(Serverless)로 동작한다.

## 2. 시스템 아키텍처 및 파이프라인

프로젝트는 크게 5단계 파이프라인으로 구성되어 GitHub Actions 환경에서 순차적으로 실행된다.

1. **데이터 수집 (Data Fetching)**:
   - **중식**: NEIS 오픈 API를 통해 당일 데이터 추출
   - **석식**: 학교 홈페이지 게시판을 크롤링하여 월간 엑셀 파일을 다운로드 및 로컬 캐싱하여 추출
2. **UI 렌더링 (Headless Rendering)**: 가상 디스플레이 환경(XVFB)에서 **Flutter의 모든 UI 위젯(Container, Text, BoxDecoration, 그림자 효과 등)을 사용하여 예쁜 식단표 화면을 메모리 상에 구성**
3. **이미지 캡처 (Image Export)**: 디자인된 위젯 화면을 스크린샷 캡처하여 `.png` 형태의 고화질 이미지 파일로 저장
4. **음악 추가 및 영상 변환 (Video Rendering)**: (선택 사항) 저장된 `.png` 이미지와 준비된 `.mp3` 음악 파일을 `FFmpeg`를 이용해 15초 길이의 `.mp4` 영상으로 변환 (인스타그램 공식 API는 '음악 스티커'를 지원하지 않으므로 영상 자체에 음악을 합성)
5. **인스타그램 업로드 (Auto Upload)**: **공식 Instagram Graph API**를 통해 정지 위험 없이 안전하게 캡처된 이미지(또는 영상)를 인스타그램 스토리에 게시

## 3. 핵심 기술 스택

### 3.1. 데이터 수집 및 렌더링 (Dart/Flutter)

- **중식 API**: `http` 패키지 (NEIS API 호출)
- **석식 크롤링**: `html` 패키지 (게시판 파싱) 및 `excel` 패키지 (엑셀 파싱)
- **UI 캡처**: `screenshot` 패키지 (위젯 트리를 캡처하여 이미지 변환)

### 3.2. GitHub Actions (CI/CD)

- **스케줄러**: Cron 기반 매일 자동 실행
- **핵심 유틸리티**:
  - `xvfb`: 화면이 없는 리눅스 서버에서 Flutter 위젯을 그려내기 위한 가상 모니터
  - `ffmpeg`: (음악 추가 시) 이미지와 음원을 합성하여 동영상 스토리로 변환

### 3.3. 인스타그램 업로드 (Python 또는 Dart)

- **안전한 업로드 방식**: **Instagram Graph API (공식 API)**
  - 비공식 봇 라이브러리는 계정 정지 위험이 매우 크므로, 메타(Meta) 공식 API를 통해 업로드.
  - _참고_: 공식 API는 인스타 앱 내부의 '음악 스티커' 부착을 지원하지 않습니다. 따라서 음악을 넣고 싶다면 4단계의 FFmpeg 합성 방식을 사용해 비디오 형태로 업로드해야 합니다.

## 4. 모듈별 기능 명세

### A. 급식 데이터 수집 모듈 (Dart)

- **중식 (NEIS API)**:
  - 나이스 API에 학교코드와 날짜를 전달하여 당일 중식 메뉴 수집.
- **석식 (웹 크롤링 및 엑셀 파싱)**:
  - 타겟 URL: `https://school.use.go.kr/ulsan-hs-h/M01030701/`
  - 로직: 특정 폴더에 이번 달 엑셀 파일이 있는지 확인.
    - 없다면 (월초 혹은 데이터 소진 시): 게시판 HTML을 파싱하여 이번 달/연도에 해당하는 엑셀 파일 첨부 게시글을 찾아 다운로드 및 저장.
    - 있다면: 저장된 엑셀 파일을 읽어와 오늘 날짜에 해당하는 석식 데이터 파싱.

### B. 위젯 렌더링 및 이미지화 모듈 (Dart/Flutter Test)

- **기능**: 수집된 데이터를 바탕으로 앱 화면 짜듯이 `1080x1920` (스토리 규격)의 이쁜 Flutter 위젯을 디자인.
- **실행 방식**: `flutter test` 환경에서 위젯을 메모리에 올리고 `screenshot` 패키지로 찍어냄.

### C. 오디오 합성 모듈 (Bash/FFmpeg)

- **기능**: 생성된 `meal_today.png`와 BGM 파일 `bgm.mp3`를 결합하여 `meal_today.mp4` 비디오 생성.

### D. 자동 업로드 모듈 (Python/Requests)

- **기능**: GitHub Secrets에 저장된 공식 API Token을 받아 공식 Graph API 엔드포인트로 사진(또는 영상) 전송 및 스토리 발행.
